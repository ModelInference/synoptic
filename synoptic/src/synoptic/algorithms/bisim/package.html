Provides for implementations of a coarsening and an refinement algorithm.

Coarsening and Refinement

Graphs can be transformed using two algorithms -- coarsening and refinement. Coarsening is implemented by Bisimulation.mergePartitions, which is an invariant-preserving version of k-Tail. It can only be run on PartitionGraphs, because Partitions can be merged and MessageEvents cannot be merged.

Refinement is provided by Bisimulation.refinePartitions which implements the BisimH algorithm. The class Bisimulation provides options via static member variables to enable experimental features: incommingEdgeSplit enables splits according to incoming edges, and interleavedMerging adds a frequent coarsening phases to Bisimulation.refinePartitions. Again, the argument must be of type PartitionGraph because Partitions can be split, but MessageEvents cannot.
