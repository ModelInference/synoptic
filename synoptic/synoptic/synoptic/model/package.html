Provides basic data structures, such as graphs, transitions, and events.

Graphs

The main data-structure in Synoptic is a graph. In the implementation a graph is represented by a set of nodes. Synoptic's graphs do not contain edges themselves, but leave it to the nodes to store the edges. Edges of nodes can be accessed via the interface INode which all classes representing nodes must implement.

There are two important classes that implement the interface INode and thus can be used to build graphs: MessageEvent and Partition. MessageEvent is used to build input graphs from raw data. This class provides methods to add edges between different nodes of type MessageEvent. A typical input graph has type Graph<MessageEvent>.

A Partition is a set of MessageEvent, and plays the role of a node in the PartitionGraph. Partitions support merging and splitting of nodes, and do not store their edges explicitly. Thus they have no direct means of inserting an edge. Instead, they generate edges implicitly and on the fly from the MessageEvents they contain. This has the advantage that edges are always up-to-date and MessageEvents can be moved freely among Partitions (as long as they are contained in at most one partition at every given time).
